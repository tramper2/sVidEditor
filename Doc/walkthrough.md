# sVidEditor 개발 완료 결과 보고서 (Walkthrough)

로컬 렌더링 방식의 웹 동영상 편집기 **sVidEditor**의 핵심 기능 개발을 완료하고 로컬 Git 저장소 구성을 마쳤습니다. 

개발된 소스 코드 파일들은 **`D:\Study\WebPage\sVidEditor`** 폴더에 배치되었으며, 웹 브라우저에서 편리하게 더블 클릭하여 즉시 실행이 가능하도록 모듈 로더 의존성을 제거하여 개발되었습니다.

---

## 1. 프로젝트 파일 목록 및 구조

프로젝트 폴더 내 구성 요소와 역할은 다음과 같습니다.

*   **index.html**: 미디어 라이브러리, 프리뷰 플레이어, 트랙 타임라인, 클립 속성 설정 창 및 FFmpeg 명령어 콘솔 창을 배치한 싱글 페이지 애플리케이션 마크업입니다.
*   **style.css**: Outfit 폰트를 활용한 다크 테마 UI 스타일시트입니다. 글래스모피즘 스타일 카드와 스크러빙이 가능한 정밀한 눈금자, 트랙 레인 및 조작 핸들 등을 정의합니다.
*   **app.js**: 타임라인 데이터의 상태 관리(State), 미디어 파일 임포트, 재생 제어, 캔버스 기반의 복합 실시간 드로잉 및 드래그 앤 드롭 편집 로직을 총괄하는 메인 엔진입니다.
*   **ffmpeg-generator.js**: 사용자의 타임라인 상태값을 읽어서 비디오 결합, 오디오 믹싱, 자막 합성 및 PIP 배치가 통합된 복잡한 Windows FFmpeg 필터 체인 명령어와 실행 배치 파일(`.bat`)을 실시간 컴파일하는 핵심 라이브러리입니다.
*   **ffmpeg/README.txt**: 로컬 렌더링에 필수적인 `ffmpeg.exe` 및 `ffprobe.exe` 바이너리를 다운로드하고 저장해야 할 위치를 설명하는 가이드라인 문서입니다.
*   **.gitignore**: 대용량 영상 리소스 및 실행 파일, 생성된 임시 스크립트 등이 깃허브에 커밋되어 충돌을 일으키지 않도록 사전에 방지하는 룰셋입니다.
*   **deploy.sh** (Bash용) / **deploy.bat** (Windows CMD용): `git@github.com:tramper2/sVidEditor.git` 저장소의 `gh-pages` 브랜치로 자동 푸시하여 배포해 주는 실행 스크립트 파일입니다.

## 2. 구현된 핵심 기능의 기술적 세부사항

### ① 이 기종 포맷(Codec 및 FPS) 영상 결합 대응
입력되는 영상들이 각각 다를 경우(예: 30fps, 60fps, 120fps, 180fps 및 서로 다른 해상도) FFmpeg의 일반 결합 명령어는 충돌이 발생합니다. 이를 위해 컴파일 시 각 영상 인풋 필터 체인의 시작 지점에 규격화 필터인 `scale=1280:720, pad=1280:720:(1280-iw)/2:(720-ih)/2:black, setsar=1, fps=60`을 일괄 적용하여 검은 레터박스로 여백을 채우고 프레임을 60fps로 표준화하여 원본 손상 없이 자연스럽게 이어지도록 했습니다.

### ② 클립 효과 프리뷰 및 드래그 앤 드롭
*   효과 라이브러리의 아이콘들을 타임라인 클립 상단에 끌어다 놓으면(HTML5 Drag & Drop API 사용) 즉시 클립 상태 필터 배열에 적용됩니다.
*   캔버스 프리뷰 시, CSS Filter (`grayscale`, `sepia`), 캔버스 변환 매트릭스(`ctx.rotate`, `ctx.scale`)를 적용하여 흑백, 세피아, 회전, 줌인/줌아웃을 브라우저 상에서 부드럽게 미리 볼 수 있게 구현했습니다.

### ③ 타임라인 줌 및 정밀 트리밍 (Trimming)
*   우측 상단의 줌 슬라이더를 통해 눈금자 크기(`px/s`)를 즉각 변경하여 타임라인 폭을 넓히거나 좁힐 수 있습니다.
*   타임라인 클립 양 끝단에 마우스를 올리면 트리밍 핸들(`trim-handle`)이 나타나며, 마우스 드래그를 통해 시작점(`sourceStart`)과 종료점(`sourceEnd`), 지속시간(`duration`)을 자유롭게 줄이고 늘릴 수 있습니다.

### ④ 다중 오디오 트랙 및 지연 믹싱
*   비디오 트랙1과 트랙2의 고유 오디오 볼륨을 각각 조절할 수 있습니다.
*   오디오 트랙(배경음악)에 삽입된 오디오 리소스들은 시작 시간 차를 맞추기 위해 FFmpeg의 `adelay=밀리초|밀리초` 필터가 자동 계산되어 적용되며, 최종적으로 `amix` 필터에 의해 메인 영상 길이에 정확하게 동기화되어 결합됩니다.

### ⑤ 자막 및 PNG 오버레이 / PIP 지원
*   **자막 추가**: 타임라인 재생 헤드 위치에 자막 클립을 생성하고 텍스트 내용, 글자 크기, 색상, 좌표를 우측 패널에서 정밀 제어할 수 있습니다.
*   **PIP (화면 속 화면)**: Video Track 2에 영상을 추가하면 자동으로 메인 영상 위에 작게 겹치는 PIP 모드가 활성화되며 크기(가로/세로) 및 위치(X, Y)를 지정해 윈도우 렌더링에 반영되게 설계되었습니다.

## 3. 최종 사용자 실행 및 검증 가이드

### 단계 1: 미디어 파일 추가 및 로컬 경로 지정
1. `D:\Study\WebPage\sVidEditor\index.html` 파일을 웹 브라우저(Chrome, Edge 등)로 엽니다. (CORS 보안 오류가 없으므로 더블 클릭만으로 바로 열립니다.)
2. 좌측 상단 드롭존을 클릭해 편집하려는 동영상과 배경 오디오 파일들을 선택합니다.
3. 타임라인에 드래그하거나 미디어 리스트를 더블클릭하여 트랙에 적절히 배치합니다.
4. **[중요]** 우측 속성 창에 표시되는 **'로컬 전체 경로'**가 해당 파일이 위치한 실제 PC 경로(예: `D:\Study\WebPage\sVidEditor\source\video1.mp4`)와 동일한지 확인하고 다를 경우 올바르게 수정해 줍니다. (FFmpeg 렌더링 시 이 경로를 참조하여 로드합니다.)

### 단계 2: 렌더링 배치 파일 다운로드 및 실행
1. 웹 편집창에서 자막 배치, 효과 지정 및 볼륨 조절을 마치면 하단에 완성된 FFmpeg 명령어가 실시간 표시됩니다.
2. 우측 상단의 **`렌더링 배치파일 내보내기`** 버튼을 누릅니다.
3. 다운로드된 `render.bat` 파일을 프로젝트 루트 폴더(`D:\Study\WebPage\sVidEditor`)에 넣습니다.
4. `ffmpeg.exe`와 `ffprobe.exe`를 `D:\Study\WebPage\sVidEditor\ffmpeg\` 폴더 안에 다운로드하여 넣어 줍니다. (시스템 PATH에 잡혀있으면 생략 가능)
5. `render.bat`을 더블 클릭하여 실행합니다. 자동으로 영상 결합 및 필터 인코딩이 동작하며 최종 완성본이 `D:\Study\WebPage\sVidEditor\output\` 폴더에 생성됩니다.

## 4. Git 및 GitHub Pages 배포 방법

프로젝트에 Git 및 원격 주소 설정이 완료되어 있습니다. 
웹 사이트를 GitHub Pages에 배포하려면 아래 둘 중 하나의 방법을 선택하여 원클릭 배포가 가능합니다.

*   **방법 A (Windows 환경 추천)**: 
    *   `D:\Study\WebPage\sVidEditor` 폴더에서 **`deploy.bat`** 파일을 더블 클릭하여 실행합니다. 
    *   스크립트가 자동으로 변경 사항을 커밋하고 `git@github.com:tramper2/sVidEditor.git`의 `gh-pages` 브랜치로 강제 푸시를 수행합니다.
*   **방법 B (Git Bash / 터미널 환경)**:
    *   터미널에서 `./deploy.sh` 명령어를 실행하면 배포 프로세스가 수행됩니다.
