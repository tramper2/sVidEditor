# 구현 계획서 - 비디오 속도 조절 기능 (0.1x ~ 6.0x)

이 계획서는 웹 기반 비디오 에디터의 메인 비디오 트랙(Video Track 1) 및 PIP 오버레이 트랙(Video Track 2) 클립의 재생 속도를 0.1배속부터 6.0배속까지 개별 조절할 수 있도록 하는 기능에 대한 명세입니다. 속도 변경 시 타임라인 상의 클립 길이가 유기적으로 변경되며, 브라우저 실시간 프리뷰 및 로컬 FFmpeg 렌더링 명령어에도 정상적으로 속도가 반영되도록 구현합니다.

## 사용자 검토 필요 사항

> [!IMPORTANT]
> **주요 설계 의사결정 사항:**
> 1. **기본 배속 및 범위**: 모든 비디오 클립은 기본적으로 `1.0x` 배속을 가집니다. 속도 조절 범위는 슬라이더 기준으로 `0.1x`부터 `6.0x`까지이며 `0.1x` 단위로 세밀하게 조절할 수 있습니다. 추가로 자주 쓰이는 배속(`0.5x`, `1.0x`, `2.0x`, `4.0x`)을 빠르게 지정할 수 있는 프리셋 버튼을 제공합니다.
> 2. **타임라인 클립 길이의 동적 연동**: 배속이 변경되면 사용자가 지정했던 원본 영상의 컷 구간(`sourceStart`, `sourceEnd`)은 유지된 채로 타임라인 길이($D_{timeline}$)가 다음과 같이 자동으로 변경됩니다: $D_{timeline} = (sourceEnd - sourceStart) / speed$.
> 3. **트리밍 핸들 조작 시 배속 적용**: 타임라인에서 클립의 왼쪽/오른쪽 트리밍 핸들을 드래그할 때 마우스 이동 거리(시간)에 클립의 속도 배율을 곱하여 원본 영상 구간의 타임스탬프를 올바르게 계산합니다.
> 4. **FFmpeg 명령어 생성 (오디오 멀티-템포 체이닝)**:
>    - 비디오: `setpts=PTS/speed` 필터를 적용합니다.
>    - 오디오: FFmpeg의 `atempo` 필터는 `0.5` 이상 `2.0` 이하만 지원합니다. 따라서 배속이 이 범위를 벗어나는 경우(예: `6.0x`), 다중 `atempo` 필터를 자동으로 체이닝하여 구현합니다 (예: `6.0x` -> `atempo=2.0,atempo=2.0,atempo=1.5`).

### 핵심 비즈니스 로직 수정

#### [MODIFY] [app.js](file:///d:/Study/WebPage/sVidEditor/app.js)
1. **상태 초기화 및 저장**:
   - `addAssetToTimeline()` 호출 시 클립의 기본 재생 속도인 `speed: 1.0`을 부여합니다.
   - `loadProjectFile()`에서 프로젝트를 불러올 때 배속 속성이 없는 기존 파일의 클립에 기본값 `1.0`을 할당하는 안전장치를 추가합니다.
2. **DOM 캐싱**:
   - 속도 입력 슬라이더(`propSpeed`), 속도 라벨(`propSpeedLabel`), 프리셋 버튼들을 DOM 캐시에 추가합니다.
3. **속성 패널 이벤트 연동**:
   - 배속 슬라이더의 `input`/`change` 이벤트 및 프리셋 버튼 클릭 이벤트를 추가합니다.
   - `selectClip()` 함수에서 클립 선택 시 배속 슬라이더와 수치 라벨을 현재 클립 속도에 맞추어 연동합니다.
   - `updateSelectedClipFromInputs(e)` 및 `applyPropertiesChanges()` 내에서 배속 변화에 맞춰 다음의 수학적 일관성을 유지하도록 수정합니다:
     - **배속이 변경된 경우**: 원본 구간 고정, 타임라인 지속시간(`duration`)을 계산 및 업데이트합니다.
     - **타임라인 지속시간이 변경된 경우**: 시작 지점 고정, 원본 종료 지점(`sourceEnd`)을 계산 및 업데이트합니다.
     - **원본 시작/종료 지점이 변경된 경우**: 타임라인 지속시간을 재계산합니다.
4. **트리밍 핸들 조작**:
   - 왼쪽/오른쪽 트리밍 핸들 드래그 시 이동 거리를 속도로 나누거나 곱하여 원본 영상 구간의 시간을 정확하게 산출하도록 변경합니다.
5. **타임라인 클립 배속 라벨 추가**:
   - 타임라인 클립 텍스트 구성 요소 내에 현재 배속이 표시되도록 연동합니다.
6. **프리뷰 및 재생 동기화**:
   - Canvas에 프레임을 그리는 `drawVideoClip()` 및 스크러빙 시 시간을 동기화하는 `syncHiddenPlayersTime()`에서 경과 시간에 클립 배속을 반영하여 정확한 영상 위치(`sourceStart + clipElapsed * speed`)를 찾아내도록 수정합니다.
   - `updateActivePlayersStates()` 루프에서 재생 중일 때 비디오 재생 엘리먼트 자체의 `playbackRate`를 클립 배속과 동일하게 동기화하고 오차 보정 시 속도 배율을 반영합니다.

### FFmpeg 스크립트 생성 로직 수정

#### [MODIFY] [ffmpeg-generator.js](file:///d:/Study/WebPage/sVidEditor/ffmpeg-generator.js)
1. **템포 필터 연쇄 처리 함수 구현**:
   - `0.1`배속부터 `6.0`배속까지 FFmpeg 한계를 우회하며 배속을 안전하게 곱해서 표현하는 `getAudioTempoFilter(speed)` 헬퍼 함수를 추가합니다.
2. **입력 파일 트리밍 기간 수정**:
   - FFmpeg에 들어갈 때 `-t ${clip.duration}` 대신 속도가 반영되지 않은 순수 원본 컷 크기인 `-t ${clip.sourceEnd - clip.sourceStart}`를 읽도록 입력 매핑부를 수정합니다.
3. **비디오 및 오디오 배속 필터 적용**:
   - Video Track 1 및 Track 2의 비디오 필터 복합체 체인에 `setpts=PTS/speed` 필터를 추가합니다.
   - 오디오 필터 복합체 체인에 체이닝된 `atempo` 필터 문자열을 추가하여 소리가 속도에 비례해 고르게 재생되도록 합니다.

## 검증 계획

### 수동 기능 테스트
1. **UI 동작 테스트**:
   - 비디오 파일을 가져와 Video Track 1에 배치합니다.
   - 클립을 선택한 뒤 속도 조절 슬라이더를 `2.0x`로 설정하고 타임라인 클립 길이가 절반으로 줄어드는지 확인합니다.
   - 프리셋 버튼 `0.5x`를 클릭하고 타임라인 클립 길이가 기존 대비 두 배로 늘어나는지 확인합니다.
   - 클립 위에 배속 배지(예: `2.0x`)가 표시되는지 확인합니다.
2. **프리뷰 재생 테스트**:
   - 배속이 변경된 상태로 타임라인을 재생시킵니다.
   - 브라우저 실시간 프리뷰가 해당 배속 속도에 맞추어 정상적으로 빠르게/느리게 재생되며 소리와 화면의 싱크가 유지되는지 확인합니다.
3. **클립 분할 테스트**:
   - `2.0x`로 배속이 설정된 비디오 클립을 중간 지점에서 분할(Split, 단축키 S)합니다.
   - 분할된 두 개 클립 모두 속도가 `2.0x`로 유지되고 싱크가 일치하는지 검토합니다.
4. **생성된 FFmpeg 명령어 테스트**:
   - 내보내기 텍스트 상자에 생성된 FFmpeg 명령어의 비디오/오디오 필터를 분석합니다.
   - 비디오: `setpts=PTS/2.0` 형태가 들어가 있는지 점검합니다.
   - 오디오: `atempo`가 배속 값에 맞게 연산(6배속 시 `atempo=2.0,atempo=2.0,atempo=1.5`)되어 명령어로 구현되는지 확인합니다.
